Contenido mínimo

Qué debe incluir la web

La web no debe parecer una demostración artificial. Debe explicar una investigación propia, con un problema concreto, una solución comprensible y límites explícitos.

01

Problema

Qué situación se quiere mejorar, a quién afecta y por qué merece la pena dedicarle una solución con IA.

02

Usuario objetivo

Quién usaría la solución, en qué contexto trabaja y qué necesita resolver de forma sencilla.

03

Solución

Cómo se aplicaría la IA para generar, organizar, buscar, analizar o automatizar parte del proceso.

04

Flujo de trabajo

Qué datos entran, qué hace la herramienta, qué resultado sale y dónde interviene la revisión humana.

05

Herramientas

Qué aplicaciones se han usado para pensar, redactar, crear imágenes, generar código o revisar errores.

06

Demo

Un ejemplo pequeño, visible y fácil de entender que demuestre cómo funcionaría la solución.

07

Beneficios

Qué mejora aporta: tiempo, claridad, orden, reducción de tareas repetitivas o mejor acceso a información.

08

Limitaciones

Qué no resuelve, qué errores puede cometer y por qué sigue siendo necesaria la validación humana.

09

Conclusión

Qué se ha aprendido, qué se podría mejorar y cuál sería el siguiente paso si el proyecto continuara.

Guion recomendado

Estructura mínima recomendada

El objetivo es que cualquier persona pueda abrir la web y entender el proyecto sin explicación previa. Cada bloque debe responder a una pregunta concreta.

01

Portada

Título del proyecto, autor, subtítulo y una idea clara de lo que se va a presentar.

02

Problema real

Descripción breve del problema, personas afectadas y motivo por el que tiene sentido resolverlo.

03

Solución propuesta

Explicación de la solución con IA, sin prometer resultados que no se hayan probado.

04

Público objetivo

Perfil de usuario, necesidad principal y situación en la que usaría la herramienta.

05

Cómo funciona

Pasos básicos: entrada de información, proceso con IA, resultado y revisión.

06

Herramientas usadas

Listado de herramientas y uso real que se les ha dado durante el proyecto.

07

Ejemplo o demo

Caso sencillo que permita ver el valor de la solución en condiciones realistas.

08

Riesgos y limitaciones

Errores posibles, datos necesarios, alcance no cubierto y necesidad de supervisión.

09

Autor

Nombre del alumno, curso, fecha y contacto o enlace si procede.

Caso de ejemplo

Modelo para alumnos

Ejemplo de proyecto

Este ejemplo muestra cómo presentar una web sencilla que explique el proyecto final, sus decisiones principales y el uso real de IA.

Proyecto
Web explicativa para presentar el proyecto final de IA aplicada.
Problema
El alumnado necesita una forma clara de ordenar su idea, justificarla y publicarla sin convertir el trabajo en algo demasiado complejo.
Solución
Crear una web estática con secciones mínimas: problema, usuario, solución, funcionamiento, herramientas, limitaciones, demo y evaluación.
Usuario
Alumno que debe entregar un proyecto final publicable y explicarlo en pocos minutos.
Beneficio
Facilita una entrega terminable, comprensible y evaluable desde un enlace público.

Rúbrica

Cómo se evalúa el proyecto

La evaluación valora que el proyecto sea comprensible, publicable y conectado con un uso real de IA. No se premia la complejidad si no aporta claridad.

15%

Definición del problema

El alumno explica qué problema quiere resolver, a quién afecta y por qué merece la pena solucionarlo.

15%

Propuesta de solución con IA

Describe cómo usaría IA para atención al cliente, contenido, organización, búsqueda, automatización o análisis.

20%

Estructura y contenido de la web

La web incluye problema, público objetivo, solución, beneficios, funcionamiento, límites y contacto o demo.

15%

Diseño visual y claridad

La web se entiende rápido, tiene buen orden visual, textos comprensibles y no está saturada.

15%

Uso práctico de herramientas de IA

Se valora el uso de IA para redactar, estructurar, mejorar imágenes, generar código, revisar errores o preparar la presentación.

10%

Publicación funcional

La web está publicada en GitHub Pages, Netlify, Vercel o similar, y se puede abrir desde un enlace.

10%

Presentación final

Explican en 3-5 minutos qué han hecho, qué problema resuelve, qué han aprendido y qué mejorarían.

Criterio mínimo para aprobar

Para aprobar, el proyecto debe tener como mínimo una web visible mediante enlace, un problema claramente definido, una propuesta de solución comprensible y una explicación básica del uso de IA.

Stack recomendado

Tecnologías recomendadas

Estas herramientas priorizan publicación rápida, edición sencilla y una curva de aprendizaje asumible para un proyecto final.

GitHub Pages

Publicación gratuita desde un repositorio, con enlace compartible para la evaluación.

HTML

Estructura básica de la página y contenido editable por secciones.

CSS

Diseño visual, espaciado, tarjetas y adaptación a móvil.

JavaScript

Interacciones simples como navegación, menú móvil y enlace activo.

ChatGPT

Apoyo para idear, redactar, revisar, sintetizar y generar código base.

No-code opcional

Herramientas complementarias para prototipos, formularios, imágenes o automatizaciones.

Criterios de calidad

Buenas prácticas de entrega

Texto claro

Evitar frases genéricas. Explicar el caso con palabras que entienda un usuario real.

Menos es más

Una web pequeña, completa y coherente vale más que una página llena de funciones sin probar.

Transparencia

Indicar qué ha hecho la IA, qué ha revisado el alumno y qué queda pendiente.

Diseño simple

Usar espacios, jerarquía visual y secciones breves para facilitar la lectura.

Caso real

Mostrar un ejemplo de entrada y salida que ayude a imaginar el uso práctico.
